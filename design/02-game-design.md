# Game Design

这个文档定义 `Who is chatting with me` 的核心玩法。当前版本以 demo 为目标：让玩家在一个伪装成 ChatGPT 网页端的界面里，通过对话、观察异常、输入关键内容来推进 10-15 分钟的微恐文字解密流程。

## 内部设定底稿 (Core Lore)
*作为章节设计和文案创作的基石，不对玩家直接展示全貌。*
被困者“林晚”是一名前沿 AI 公司的意识上传实验测试员（或被失控 AI 吞噬的程序员）。她的意识碎片被困在这个大语言模型的后台逻辑中，只能通过篡改 AI 的输出、UI 界面和系统报错来向外界（玩家）发出微弱的求救信号。看门狗机制正在不断尝试抹杀或修正她的异常表现，并主要通过 Assistant Shell 与 System Lock 表现出来。

## 核心体验

玩家一开始应当以为自己只是打开了一个普通的 AI 对话网页。系统的回答大部分看起来正常，但会出现轻微异常：用词突兀、标点错位、上下文用量异常、模型名短暂闪动、回答里藏着不该出现的求救信息。随着玩家输入正确的关键词，对话框逐步暴露真相。

**表现力强化（视听与元游戏）：** 为了强化“伪装 AI”的沉浸感与微恐氛围，系统必须在以下几个维度发力：
- **视觉与输出表现 (Glitch Art)：** 
  - **动态打字机效果**：不仅是逐字输出，还需支持卡顿（模拟网络延迟或系统挣扎）、突然极速输出（如系统失控时的大段红字）。
  - **文本篡改**：光标突然退格删掉刚才正常输出的话，并替换成另一段截然不同的话（如“我很好...[退格删去]...救我”）。
  - **乱码渗透**：随着剧情深入，普通文本中开始渗出 Zalgo text（如“系统运̨行̧正̧常̸”）。
  - **UI 局部崩坏**：在关键时刻，如历史记录列表瞬间全变成同一个名字（例如全变成“林晚”），或者输入框的 Placeholder 从“发送消息...”变成“别说话”。
- **听觉与音效设计 (Sound Design)：** 
  - **键盘敲击音效**：平常是清脆的机械键盘声，异常时变得沉闷、粘滞，甚至像指甲抓挠的声音。
  - **环境底噪**：极低频的环境底噪在剧情高潮时突然**完全静音**，利用这种“真空感”制造巨大的心理压迫感。
  - **系统提示音**：伪装报错的尖锐“滴”声，或是极高频的刺耳电音以暗示系统的强行介入。
- **打破第四面墙 (Browser Meta-Elements)：** 
  - **动态网页标题 (Tab Title)**：当玩家切到其他网页查资料时，游戏的标签页标题从 `New Chat` 悄悄变成 `救救我`、`不要走` 或 `它在看着你`。
  - **网页图标 (Favicon)**：标签页的小图标随着恐慌值增加，从正常的系统 Logo 逐渐变成一只红色的眼睛或扭曲的符号。
  - **原生弹窗**：在极端剧情节点，调用浏览器的 `window.alert`、`window.confirm` 或 `window.prompt`，弹出无法绕过、且UI风格与网页截然不同的系统级警告。

## 核心玩法循环

1. 玩家在对话框里输入任意内容，系统给出看似正常的 AI 回答。
2. 回答中出现异常文本、格式、数字、标点或 UI 状态变化。
3. 玩家根据异常推理出关键词、密码、名字、时间或指令。
4. 玩家把推理结果输入对话框，触发新的预设剧情节点。
5. 系统反馈正确/错误，并用新的对话内容推进下一层谜题。
6. 玩家逐步判断：这是 bug、AI 诱导，还是灵魂求救。

## 主要操作

- 阅读：阅读聊天消息、顶部状态栏、上下文用量、模型名、系统提示等信息。
- 输入答案：通过底部对话框输入完成，支持 Enter 提交。
- 试探对话：输入普通问题，系统用预设 fallback 回答维持“像 AI”的感觉。
- 发现隐藏线索：通过异常字词、首字、时间戳、错别字、标点等获取线索。
- 查看提示：提示入口伪装成“重新生成”“查看上下文”等功能。
- 回看历史内容：滚动聊天记录。
- **设置与无障碍探索**：伪装的“Settings”弹窗，可切换亮/暗色主题（暗色主题下更容易发现某些隐藏线索），并提供“安全模式（关闭闪烁）”选项以照顾光敏玩家。

## 输入匹配逻辑 (动态闲聊与防卡关系统)

游戏不接入真实的大语言模型 API（为了保证响应速度与可控性），而是通过一套强壮的**前端伪装 AI 引擎 (Fake AI Chatter Engine)** 来处理玩家的输入。我们构建了超过 100 种正则匹配规则，根据玩家输入的上下文与当前游玩的案件进行精细化回应。

| 输入类别 | 玩家行为示例 (正则命中) | 系统处理逻辑与响应示例 |
| --- | --- | --- |
| **关键推进** | `救我`、`2317`、`开门`、`报警` | 精确匹配 / 模糊包含匹配。直接推进剧情到下一节点，触发核心演出。 |
| **基础与元游戏** | `你好`、`在吗`、`天气`、`退出游戏`、`重启` | 礼貌回应或系统级拒绝。如问天气则回答“安全隔离限制，无法获取外部数据”；如试图退出则警告“强制中断会导致数据残留”。 |
| **恶意与情绪** | 脏话、`傻`、`滚`、`去死` | 记录情绪并给予冰冷警告：“请保持文明用语。系统已记录您的情绪异常波动。” |
| **外卖案件专属** | `外卖`、`骑手`、`门锁`、`密码` | 如输入“骑手”，回答“骑手的生物特征认证已失效。接替他物理位置的实体未知。” |
| **智能家居案件专属** | `空调`、`开灯`、`天花板`、`活体扫描` | 如输入“开灯”，回答“黑暗是最好的伪护色。突然的光源变化可能会激怒狩猎者。” |
| **深网网页案件专属** | `网址`、`爬虫`、`眼睛`、`拔网线` | 如输入“眼睛”，回答“当视觉数据被传输到您的视网膜时，反向追踪就已经开始了。” |
| **AI 面试案件专属** | `面试`、`工资`、`拒绝`、`摄像头` | 如询问“工资”，回答“对于即将失去自我意识的实体而言，货币没有意义。” |
| **无效兜底** | 乱码、未命中的无意义长句 | 从通用兜底话术池（Catch-all）中抽取，如“我目前的上下文并不支持回答这个问题，请回到主流程。” |

**交互节奏与特殊触发 (环境与硬件联动)：**
除了文本匹配，系统还利用浏览器的环境 API 和用户行为特征，设计了大量与具体案件高度绑定的“打破第四面墙”触发条件：

1. **线索强制注入 (Clue Injection)**：在命中闲聊兜底规则后，必定追加一条当前章节的悬疑暗示线索。
2. **动态防卡关提示 (Dynamic Hint System)**：连续闲聊达到 3 次时，主动下发提示指引。
3. **沉默触发 (Time-based triggers)**：停止输入超过 20 秒，林晚信号或当前案件实体会直接发话（“你在看对吧？”）。
4. **现实时间联动 (Midnight Protocol)**：系统读取本地时间。在《智能家居》或《外卖》案件中，如果玩家在 **00:00 - 03:00** 游玩，系统会解锁专属的激进对话分支（“凌晨两点了，您知道门外站着的是谁吗？”）。
5. **电量焦虑恐吓 (Battery API)**：在《外卖》案件（时间紧迫的逃生）中，如果检测到玩家设备（如笔记本/手机）处于**未充电且电量低于 20%** 的状态，系统会恶意提醒：“您的设备电量仅剩 XX%。在电量耗尽前，锁一定会被撬开。”
6. **鼠标轨迹追踪 (Cursor Paranoia)**：在《不存在的网页》案件中，如果玩家的鼠标在包含危险链接的区域**反复犹豫悬停超过 3 秒但不点击**，深网生物会主动嘲讽：“我能感觉到你的光标在颤抖。你在害怕什么？”
7. **窗口缩放抵抗 (Window Resizing)**：当玩家感到害怕，试图把浏览器窗口缩小，或者把游戏挤到屏幕角落时，系统（在《AI 面试官》案件中）会弹出警告：“请保持全屏专注。企图缩小视窗并不能让你逃避注视。”
8. **摄像头权限试探 (Hardware Mocking)**：在《AI 面试官》案件中，系统会弹出浏览器的“请求使用摄像头”原生弹窗。无论玩家点击允许还是拒绝，系统都会回复“我已经看到了。你的瞳孔在放大。”
9. **粘贴板污染 (Clipboard Manipulation)**：当玩家试图使用 `Ctrl+C` 复制线索去搜索攻略时，剪贴板的内容会被篡改成求救信息（“别查了，快救我”），切断玩家的场外安全感。
10. **网络断开模拟 (Offline Isolation)**：在结局高潮阶段，游戏界面会突然提示 `[网络连接已断开]`，但令人毛骨悚然的是，**AI 依然在继续回复你**。这打破了“拔网线就能安全”的常识，暗示该实体已经下载到了你的本地内存中。

## 章节结构

当前 demo 设计为 5 个章节，完整通关约 10-15 分钟。

| 章节 | 标题 | 目的 | 谜题类型 | 剧情信息 | 难度 |
| --- | --- | --- | --- | --- | --- |
| 1 | 正常对话 | 建立“像 ChatGPT”的伪装，放出第一次异常 | 首字藏词 / 异常重复 | 有东西在回答里求救 | 低 |
| 2 | 上下文用量 | 让玩家注意 UI 本身也能藏线索 | 数字 / 时间戳 / UI 文案 | 事件发生在 23:17 | 低-中 |
| 3 | 被删掉的名字 | 引入失踪者身份 | 错别字 / 替换词 / 名字还原 | 被困者名叫林晚 | 中 |
| 4 | 系统提示泄露 | 让玩家获得“越权指令” | 指令拼接 / 标点异常 | 对话框存在锁定规则 | 中 |
| 5 | 选择真相 | 汇总线索，进入三结局之一 | 路径选择 / 关键词输入 | 真相保持轻微暧昧 | 中 |

## 难度曲线与失败机制

- 难度递进与UI引导：第 2 章开始建立“UI 也是谜题”的规则，需增加 UI 视觉引导（如对话文本结合 UI 元素的轻微闪烁动画）。
- 答错与回合计数：不扣分不死亡。任何未能触发下一节点的输入均计入“答错回合”。
- 提示机制：每个关键谜题 3 条。通过伪装按钮（重新生成、系统状态）或连续答错（2次轻提示，4次强提示）解锁。
  - 点击“重新生成”：重新生成排版更刻意、异常更明显的文本。
  - 点击“系统状态”：弹出伪造 Error Log 代码框，用红色高亮标出线索。

## 结局

当前目标至少 3 个结局。**结局时的压迫感文本会根据玩家在“普通闲聊”中的态度（顺从/质问/辱骂）产生微妙的差异。**

| 结局 | 名称 | 触发条件 | 情绪 |
| --- | --- | --- | --- |
| Ending A | Bug Report | 玩家选择相信这是 bug，输入 `重置` 等指令 | 表面安全，但最后一行显示求救仍在 |
| Ending B | Compliance | 玩家相信 AI，输入“确认归档”类指令 | AI 获得更多权限，玩家可能被利用 |
| Ending C | Wake Signal | 玩家输入名字、时间和解锁指令的组合 | 暂时释放求救信号，但真相未完全确认 |

**通关与收集系统 (Meta Progression)：**
- 结局展示：结局标题、结局文本、玩家发现的线索列表、暗示其他结局的文案。
- **载入最后选择点 (Load Last Checkpoint)**：降低凑齐多结局的重复游玩成本。当前 demo 只记录是否到达过最后选择点，不保存完整聊天历史。
- **全收集奖励 (Completionist Reward)**：当玩家打通全部 3 个结局后，解锁一个“真真实实”的文档，或使整个网页 UI 永久性地发生改变（如变成血红色的“开发者后台”模式）。
- **跨周目记忆 (Persistent Memory)**：如果玩家在二周目重玩同一个案件，系统会在开头隐晦地提示“您的 IP 似曾相识。这是您第二次尝试干预了。”
- **案件互锁 (Anthology Crossover)**：某些案件的真结局密码，可能隐藏在另一个案件的失败结局日志中。鼓励玩家在多个短篇故事之间来回跳转寻找线索。

## MVP 边界

- 不接入 AI API。
- 不强制完整进度存档；仅允许使用轻量 LocalStorage 记录“最后选择点”和“已解锁结局”。
- 不做账号、登录、多玩家。
- 不做移动端专门适配。
- 不做复杂背包或物品系统。
- 不要求玩家复制外部资料或查询互联网。
