# Development Plan

这个文档定义 `Who is chatting with me` 的开发顺序和验收标准。当前目标是先完成可玩的桌面端 demo。

## 里程碑

### M1 - 对话原型

目标：打开网页后，玩家可以像和 AI 对话一样输入内容，并完成第一个谜题。

- [ ] 初始化 React + Vite + TypeScript 项目
- [ ] 搭建伪 ChatGPT 桌面端基础布局
- [ ] 实现聊天消息列表和底部输入框
- [ ] 支持 Enter 提交、Shift + Enter 换行
- [ ] 接入 P001 第一条异常
- [ ] 实现答案匹配和 fallback 回复
- [ ] 实现基础 typing 延迟

验收：

- 首屏 5 秒内可开始输入。
- 玩家输入任意内容后出现第一条预设回复。
- 输入 `救我` 或别名可以推进剧情。

### M2 - 完整谜题流程

目标：完成 5 个章节和 5 个关键谜题，能从开始玩到最终选择。

- [ ] 接入 P002 上下文里的时间
- [ ] 接入 P003 被删掉的名字
- [ ] 接入 P004 锁的指令
- [ ] 接入 P005 最后的选择
- [ ] 实现 3 级提示机制
- [ ] 实现顶部上下文用量和模型名异常
- [ ] 增加每章 fallback 回复
- [ ] 完成主要剧情文案第一版

验收：

- 玩家不看开发说明也能完成一条主线。
- 每个谜题至少有一条轻提示和一条强提示。
- 错误输入不会让流程卡住。

### M3 - 三结局与打磨

目标：三种结局都可进入，界面和文案具备完整 demo 观感。

- [ ] 实现 Ending A `Bug Report`
- [ ] 实现 Ending B `Compliance`
- [ ] 实现 Ending C `Wake Signal`
- [ ] 结局页展示发现线索
- [ ] 添加重新开始
- [ ] 调整消息节奏、延迟和异常效果
- [ ] 完成桌面端视觉打磨

验收：

- 三个结局均可通过不同输入进入。
- 结局之间有明确情绪差异。
- 玩家可以重新开始体验其他结局。

### M4 - 发布准备

目标：可以发给别人试玩。

- [ ] 浏览器冒烟测试
- [ ] Playwright 跑通主流程
- [ ] Vitest 覆盖输入归一化和匹配
- [ ] 检查文本溢出和布局错位
- [ ] 写简单运行说明
- [ ] 生成可部署静态构建

验收：

- 本地启动无报错。
- 打包构建无报错。
- 桌面端 1280px 宽度下布局稳定。

## 验收清单

- [ ] 首次打开页面无报错。
- [ ] 首屏看起来像一个 AI 对话网页，而不是传统游戏主页。
- [ ] 玩家输入任意内容后能进入剧情。
- [ ] 玩家可以通过对话框完成所有关键推进。
- [ ] P001-P005 均可被正确答案和别名触发。
- [ ] 错误答案有自然反馈，不破坏沉浸。
- [ ] 每个谜题都有 3 级提示。
- [ ] 三个结局都能触发。
- [ ] 不需要联网，不调用 AI API。
- [ ] 桌面端布局稳定，文本不明显溢出。

## 建议开发顺序

1. 先做纯功能版聊天状态机，不急着打磨视觉。
2. 做出 P001，可输入任意内容触发第一条异常。
3. 加入答案匹配、别名和 fallback。
4. 扩展到 P002-P004。
5. 加入 P005 和三结局。
6. 最后做 UI 伪装、上下文闪动、模型名异常和节奏打磨。
