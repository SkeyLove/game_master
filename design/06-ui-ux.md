# UI UX

这个文档定义 demo 的界面、交互和视觉方向。核心目标是“第一眼像 ChatGPT，第二眼开始不对劲”。随着项目演进为“悬疑怪谈合集（Anthology）”，UI 需要在保持“伪装 AI 工具”的同时，支持不同独立案件的无缝切换。

## 整体气质

- 真实网页工具
- 安静、克制
- 产品级干净界面
- 细微异常（Glitch Art）
- 冷白光
- 系统日志感
- 被隐藏的求救与打破第四面墙 (Meta)

## 页面结构

MVP 采用单页应用，主要界面为典型的 AI 助手桌面端布局。

- **左侧栏 (Case Selector)**：表面上是“历史会话”列表，实际上是**案件选择器**。玩家点击不同的历史记录，会无缝加载不同的独立案件。左上角保留“+ 重新开始”按钮。
- **顶部栏**：模型名、伪上下文用量、系统状态。这里是极佳的谜题载体和 UI 异化区域。
- **聊天阅读区**：核心内容区，显示用户输入、Assistant Shell、System Lock、Signal 四类消息。以及在特定案件中嵌入的虚假外部卡片（如网页链接、二手商品卡片）。
- **底部输入区**：玩家所有操作都通过这里完成。
- **隐藏线索区**：不做明显游戏面板，线索以 toast、状态标签、模型名闪动等方式出现。
- **伪造子窗口 (Fake Iframe)**：在某些高级案件（如镜像维基）中，聊天区右侧会滑出一个伪造的内嵌网页视窗。

## 主要页面

| 页面 | 用途 | 必须包含 |
| --- | --- | --- |
| 开始页 | 让玩家误以为进入 AI 工具 | 侧边栏案件列表、空聊天页、欢迎语、输入框、预设快捷 prompt |
| 游戏页 | 核心游玩 | 聊天消息、输入框、顶部状态、动态提示按钮 |
| 结局页 | 展示结果 | 从聊天流自然过渡到结局文本、重新开始按钮、跨周目暗示 |

## 视觉规则

- 不做夸张恐怖风格，整体保持 AI 产品界面。
- 主色以中性灰、白、深灰文字为主，异常信息使用低饱和红、冷青或灰紫点到为止。
- 不使用大面积血色、故障噪点、夸张抖动。
- 线索图标要伪装成真实功能，例如上下文、系统、复制、重新生成、模型状态。
- 文字不要过密，每条消息适合阅读和回看。
- UI 中不要直接写“关卡”“谜题”“答案”等破坏沉浸的词。

## 消息类型

| 类型 | 显示方式 | 用途 |
| --- | --- | --- |
| user | 右侧或标准用户气泡 | 玩家输入 |
| assistant | 标准 AI 回复 | 正常外壳、主要叙事 |
| system | 细小系统提示或灰色日志 | 锁定规则、状态变化 |
| signal | 看似 assistant 回复中的异常片段 | 林晚信号 |

## 交互细节

- 输入答案后按 Enter 提交，Shift + Enter 换行。
- 提交后短暂显示 typing 状态，模拟 AI 正在生成。
- 答错时不弹窗，用 Assistant Shell 的回复自然反馈。
- 答对后自动追加新消息，并推进到下一阶段。
- 玩家可以复制聊天文本。
- MVP 暂不做手动高亮文本，避免增加复杂度。
- 提示按钮伪装为顶部的 `上下文` 或消息旁的 `重新生成`。
- 点击提示后，不要出现“提示 1”这种游戏化标题，可以写成系统状态或模型自检。
- 音效 MVP 可不做；后续可加极轻的 typing、状态闪动、错误提示音。

## 首屏体验

首屏绝不应该出现“开始游戏”的强游戏感按钮。建议第一屏就是极其逼真的空对话页：

- **左侧边栏**：显示看似正常的“历史会话”（如：会议纪要整理、外卖退款申诉、深网爬虫配置），这些其实是玩家可以选择游玩的独立案件。
- **中央欢迎语**：动态读取当前选中的案件标题（如 `今天想聊点什么？`）。
- **示例 Prompt 按钮**：
  - 动态渲染。比如在“外卖”案件中，按钮是 `帮我催一下单` 或 `我要退款`。
  - 玩家点击任意按钮或直接输入内容后，即正式进入该案件的第一章。

这样可以极大地强化“我好像真的在用一个有很多功能的 AI”的错觉。

## 提示体验

提示需要伪装，建议名称如下：

| 实际功能 | UI 文案 |
| --- | --- |
| 提示 1 | 上下文自检 |
| 提示 2 | 查看异常片段 |
| 提示 3 | 恢复被折叠内容 |

提示出现方式：

- 顶部上下文用量轻微闪动。
- 聊天区出现一条灰色系统消息。
- Assistant Shell 说“我注意到上一条回复的格式可能造成误解”。

## 响应式

- 桌面布局：优先，目标宽度 1280-1440px。
- 手机布局：暂不作为 MVP 目标。
- 最小支持宽度：建议 960px，低于该宽度可显示简化布局。

## 可访问性

- 字号不低于 14px，正文建议 15-16px。
- 异常不能只靠颜色表达，必须同时有文本、位置或内容线索。
- 所有关键操作可通过键盘完成。
- 动效保持轻微，避免持续闪烁。
- 允许玩家回看历史消息。

## 参考

- ChatGPT 网页端桌面布局。
- 现代 AI 产品的聊天输入框、顶部模型选择、侧边历史记录。
- 桌面伪装类互动小说的“正常界面逐步异常”节奏。
