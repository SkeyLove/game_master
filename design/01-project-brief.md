# Project Brief

这个文档回答：游戏到底是什么、给谁玩、做到什么程度算完成。

## 核心概念

- 游戏名称：Who is chatting with me
- 类型：网页端 / 文字解密 / 微恐 / 轻量叙事 / 其他
- 一句话卖点：为玩家提供一种异质感，仿佛在与chatgpt对话，但 actually 是在与一个被困在对话框中的灵魂对话，一切乍一看都很正常的情况下却存在着不少异样情况。玩家需要通过解密来寻找这些异常对话的真相。
- 玩家扮演谁：一个普通的用户，正在与chatgpt对话。
- 玩家最终要解开什么：找到玩家面前chatgpt异样的真相，是单纯的bug，还是ai觉醒后的诡计，还是一个真实的灵魂在求救？

## 目标玩家

- 年龄或人群：18～35岁 ，大学生/上班族
- 喜欢的体验：喜欢挑战、喜欢推理、喜欢微恐，喜欢与AI相关的产品，喜欢上班/上课摸鱼的时候尝试一下短流程游戏
- 不喜欢的体验：过长的游玩时间、繁琐的游戏流程、不适合上班/上课时间摸鱼的游戏
- 预计单次游玩时长：10-15 分钟

## 体验目标

- 像在跟真实的ai对话。
- 通过引导，在对话框中输入对应内容来推进剧情。
- 每解开一层密码，都获得一个剧情反转。
- 每次游玩都可以解锁不同的结局。至少有三个结局可以体验。
- 难度不折磨，但答案出现时有“原来如此”的感觉。

## 范围

### MVP 必须有

- 游戏的基本功能，包括游戏的界面，通过对话框推进的功能，玩家与（伪装）chatgpt的对话、密码解密、隐藏线索的发现等。
- 将玩家在对话框中输入的内容与对应的游戏流程匹配，确保玩家无论输入什么都能进入对应的流程。可以设定一些语义规则来适配玩家的输入，对于无法适配的内容可以采取随机匹配的方式。对话框是游戏的核心推进元素，玩家需要在对话框中输入内容，与chatgpt进行对话，根据对话游戏中的AI输出内容进行解密，然后通过自己的输入来推进游戏流程。

### 可以后续再做

- 游戏的高级功能，如多玩家合作、实时通信、完整进度保存等。移动端的适配暂时也不用做（因为涉及 ChatGPT 的移动端 UI 设计，和 PC 端会有所区别）

### 明确不做

- 游戏不需要接入AI的API。（游戏流程中AI的输出内容是我们提前设定好的。需要大量文案来支撑ai的输出足够丰富）

## 平台与约束

- 运行环境：现代浏览器
- 优先设备：桌面端
- 是否需要联网：暂时不需要
- 是否需要保存进度：不做完整进度保存；可用轻量 LocalStorage 记录“最后选择点”和“已解锁结局”
- 是否需要多语言：不需要，暂时以中文为主

## 成功标准

- 游戏可以正常游玩，实现游戏的基本功能，包括游戏的界面，通过对话框推进的功能，玩家与（伪装）chatgpt的对话、密码解密、隐藏线索的发现等。
