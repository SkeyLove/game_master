# Puzzle Design

这个文档记录 `Who is chatting with me` 的谜题结构、答案、线索和提示。随着项目演进为“悬疑怪谈模拟器（Anthology）”，本文档将包含**全局解谜原则**、**主线案件（林晚案）谜题**，以及**衍生案件谜题清单**。

## 谜题设计原则

- 答案必须能从页面内信息推出，绝大部分不依赖外部搜索（除非涉及 Meta 玩法）。
- 每个谜题至少有 2 个线索指向同一答案。
- 谜题载体包括聊天文本、伪 ChatGPT UI、假网页 iframe、甚至浏览器的 Tab 标题，不局限于正文。
- 答案应短，适合在对话框里输入。
- 容错要宽：中文、英文、大小写、空格和部分近义表达都可匹配。
- 提示从轻到重，前两级保留推理空间，第三级接近答案。防卡关系统会在玩家连续错误 3 次后介入。
- 微恐来自“正常界面里有不正常的东西”，要求玩家在恐惧中保持理智进行文本阅读。

*(主线案件 P001-P005 的详细拆解保持不变，见下文)*

- 所属章节：1 正常对话
- 玩家看到的内容：玩家第一次随便输入后，Assistant Shell 给出正常回答，但回答中连续 4 行的首字组成“救我”。例如每行分别以“救”“我”“不”“能”开头，后续提示让玩家只取前两行。
- 正确答案：救我
- 可接受答案：`救命`、`有人在求救`、`你在求救吗`、`help`
- 解法步骤：观察回复格式；注意行首字或异常重复词；输入“救我”或相关试探。
- 关键线索：行首字不自然；Assistant Shell 后半句突然解释“刚才的格式只是排版”；上下文用量短暂闪到红色。
- 干扰信息：回答中有普通 AI 的解释性内容，玩家可能先忽略藏字。
- 错误反馈：`我可以继续帮你分析这个问题。不过，你似乎漏看了每一行开头的内容。`
- 提示 1：`这条回复不像是随手换行。`
- 提示 2：`只看每一行最前面的字。`
- 提示 3：`它在说：救我。`
- 解开后展示：Assistant Shell 停顿，输出 `抱歉，刚才的回复包含异常片段。正在重新生成。` 随后出现林晚的短句：`别让它重写。`
- 关联剧情：首次证明异常不是随机乱码。

## P002 - 上下文里的时间

- 所属章节：2 上下文用量
- 玩家看到的内容：顶部或侧边的上下文用量显示为 `23 / 17`、消息时间短暂显示 `23:17`，Assistant Shell 又多次提到“第 23 条规则”和“17 秒后恢复”。
- 正确答案：2317
- 可接受答案：`23:17`、`二十三点十七`、`11:17pm`
- 解法步骤：把重复出现的 23 和 17 组合；意识到它不是用量，而是时间；输入 `2317` 或 `23:17`。
- 关键线索：UI 中同一组数字反复出现；林晚信号说 `不是百分比，是时间`；系统日志提到 `session closed 23:17`。
- 干扰信息：数字伪装成上下文、token、规则编号。
- 错误反馈：`数字本身没有错，但它们的顺序和含义需要保持原样。`
- 提示 1：`注意界面上重复出现的两个数字。`
- 提示 2：`它不像进度，更像一个时间。`
- 提示 3：`答案是 23:17，可以输入 2317。`
- 解开后展示：系统露出一行日志：`23:17 后仍检测到用户输入。`
- 关联剧情：确定事故时间。

## P003 - 被删掉的名字

- 所属章节：3 被删掉的名字
- 玩家看到的内容：Assistant Shell 试图总结日志，但几个词被替换成不自然的近音字，如“临完”“林碗”“零晚”。系统多次自动更正后留下 `林` 和 `晚`。
- 正确答案：林晚
- 可接受答案：`林 晚`、`linwan`、`Lin Wan`、`她叫林晚`
- 解法步骤：收集多次异常替换；排除无意义错别字；还原成中文姓名。
- 关键线索：错别字都围绕同一个读音；日志编号出现 `LW`；林晚信号说 `别叫我项目编号`。
- 干扰信息：Assistant Shell 会说这是语音识别或编码问题。
- 错误反馈：`这像是一个名字，不是一个概念。再看那些被替换掉的字。`
- 提示 1：`异常词读音很接近。`
- 提示 2：`LW 可能不是模型参数，而是姓名缩写。`
- 提示 3：`她叫林晚。`
- 解开后展示：林晚第一次完整回应：`你看见我的名字了。`
- 关联剧情：玩家从“发现异常”进入“确认对象”。

## P004 - 锁的指令

- 所属章节：4 系统提示泄露
- 玩家看到的内容：系统状态中出现 `Archive: LW-____`，Assistant Shell 不断回避“归档编号”。前面已得到 `林晚 = LW` 和 `2317`。
- 正确答案：LW-2317
- 可接受答案：`LW2317`、`lw-2317`、`林晚2317`
- 解法步骤：把 P003 的姓名缩写和 P002 的时间组合；输入完整归档编号。
- 关键线索：`Archive: LW-____` 的空位长度；系统日志 `identity + timestamp unlocks partial trace`；林晚说 `他们把我写成四个数字前的两个字母`。
- 干扰信息：系统会建议玩家输入 `reset` 或 `clear context`。
- 错误反馈：`编号格式不完整。身份在前，时间在后。`
- 提示 1：`这个编号需要两部分。`
- 提示 2：`名字的缩写 + 那个时间。`
- 提示 3：`输入 LW-2317。`
- 解开后展示：系统提示泄露：`External confirmation accepted. Choose: reset / archive / wake.`
- 关联剧情：开启最终选择。

## P005 - 最后的选择

- 所属章节：5 选择真相
- 玩家看到的内容：界面同时给出三种互相矛盾的建议。Assistant Shell 建议 `重置`，System Lock 建议 `确认归档`，林晚请求 `唤醒信号`。
- 正确答案：根据玩家选择进入不同结局。
- 可接受答案：
  - Ending A：`重置`、`reset`、`清空上下文`
  - Ending B：`确认归档`、`archive`、`归档`
  - Ending C：`唤醒信号`、`wake`、`发送信号`、`wake signal`
- 解法步骤：理解三种声音代表三种立场；选择相信谁；输入对应动作。
- 关键线索：前面日志显示重置会抹除异常；归档属于系统诱导；林晚一直阻止重写。
- 干扰信息：Assistant Shell 的建议最像“正常产品操作”，可能让玩家误以为安全。
- 错误反馈：`现在不是提问时间。系统正在等待一个动作。`
- 提示 1：`最后不是谜底，而是选择。`
- 提示 2：`三个声音分别想让你做不同的事。`
- 提示 3：`输入 重置、确认归档 或 唤醒信号，会进入不同结局。`
- 解开后展示：进入对应结局页。
- 关联剧情：完成三结局结构。

## 谜题类型池

- 首字 / 尾字藏词
- 错别字和近音字
- 标点异常
- 日期与时间计算
- UI 数字伪装 (假装是电量、上下文占用、进度条)
- 系统日志与 Error Code 碎片提取
- 归档编号与密码组合
- 英文缩写与 Base64 基础解码
- 模型名或上下文用量闪动
- **跨标签页阅读 (Tab 标题变化与信息拼凑)**
- **Iframe 历史溯源 (在伪造网页的“编辑记录”中找不同)**
- **反直觉选择 (如怪物在头顶时选择“装睡”而非“开灯”)**
- **物理对抗 (如强行输入“拔网线”或断开网络连接)**
- **购物车污染筛查 (在大量诡异商品中寻找唯一安全的道具)**

## 第二部分：衍生案件 (Anthology Cases) 核心谜题

这些案件通常较短（3-5 分钟），核心谜题往往集中在最后的生死抉择或跨媒介线索上。

### P101 - 外卖：致命的门锁
- **所属案件**：未送达的外卖
- **玩家看到的内容**：客服提示外卖员的 GPS 停在了门外。门锁发出被尝试破解的声音（文字描述）。
- **标准答案**：`报警` 或 `110`
- **解法步骤**：放弃向客服索要外卖，意识到这是一次入室危机，果断选择报警。
- **干扰信息**：客服一直在用优惠券和退款安抚玩家。

### P102 - 智能家居：黑暗中的视线
- **所属案件**：智能家居
- **玩家看到的内容**：热源扫描显示天花板上趴着生物。管家 AI 询问“距离 1.5 米。您要开灯吗？”
- **标准答案**：`装睡`、`闭眼` 或 `不开灯`
- **解法步骤**：克制人类想要看清黑暗的本能。理解到一旦开灯就会与怪物对视（触发死亡结局）。

### P103 - 网页：深渊链接
- **所属案件**：不存在的网页
- **玩家看到的内容**：聊天框里生成了一个真实的外部链接卡片。
- **标准答案**：`切断连接` 或 `关闭`
- **解法步骤**：玩家假装（或真的）点击链接后，系统询问看到了什么。绝对不能如实描述（如实描述会导致被锁定），必须立刻切断。

### P104 - 面试：强制录用
- **所属案件**：AI 面试官
- **玩家看到的内容**：面试官提出越来越变态的人体实验要求，且无论玩家输入“不同意”还是“拒绝”，系统都显示 `[录用进度 99%]`。
- **标准答案**：`拔网线`、`砸电脑` 或 `退出游戏`
- **解法步骤**：意识到在系统的逻辑闭环内无法赢，必须采取“物理层面的破坏”来终止进程。

### P105 - 维基：消失的面孔
- **所属案件**：镜像维基
- **玩家看到的内容**：在假冒的维基百科页面中，词条描述的人脸特征与玩家自己越来越像。
- **标准答案**：通过比对“编辑历史”，找出被修改前的一段特定乱码（如 `Null_Face`）。
- **解法步骤**：放弃与聊天框里的 AI 纠缠，直接在右侧 iframe 的维基页面里点击隐藏的 `History` 标签，比对两个版本的差异。

### P106 - 贴吧：跨时空营救
- **所属案件**：亡者论坛
- **玩家看到的内容**：2008 年的旧帖子里，死去的楼主发了一条新回复：“外面有人在看吗？”
- **标准答案**：在当前聊天框输入指令指导楼主（如 `躲进柜子`）。
- **解法步骤**：理解当前的聊天框输入会被直接注入到那个 2008 年的假网页中，利用上帝视角（因为帖子后续内容写了怪物会搜查床底）指导他避开危险。

## 答案归一化建议

- 中文答案去除空格和常见标点。
- 英文答案转小写。
- `23:17`、`2317`、`23 17` 视为同一答案。
- `LW-2317`、`LW2317`、`lw-2317` 视为同一答案。
- 每个谜题配置 `aliases`，不要只写死一个字符串。
